序列化和反序列化

1）序列化：Java对象转化为字节序列格式，方便网络传输

2）反序列化：字节序列格式转化为Java对象

不可变类

概述

一旦被定义就无法发生修改

特征

  • final修饰，防止子类继承
  • 类内部定义的所有，都通过private和final修饰
  • 那总不能空属性吧，所以通过构造函数初始化字段
  • 不提供setter
  • 如果有外部可变对象引用，为了保证无法在外部发生修改。通过new该对象创建新副本

常见不可变类：String、包装类、BigDecimal、LocalDate

优点

  • 线程安全
  • 不可变对象缓存共享安全
  • 保证状态统一

举例～String

String是一个不可变类，在我们创建一个String对象String a = "nihao"，a无法发生修改

当我们使用a + "abcdefg"，它会创建一个新String对象，存储拼接后的值，先前的a指向的还是老对象，只不过引用指向了新对象

所以不建议频繁拼接，对象一堆谁也不乐意看

探究String的本质

String本质上就是一个字符数组，final修饰，表示无法继承

但是数组内部还是可以发生改变

接着使用private修饰，不对外暴露setter方法，这样外部就真的无法修改了

当然如果有修改需求，replace方法就是一个很好的模范，需要创建新对象

replace方法无疑不是拷贝老字符数组到新字符数组里，最后创建新对象

Java中有哪些集合？

分类

集合分为两大类

  • Collection集合：存储对象集合
  • Map集合：存储键值对集合

Collection接口分为List、Set、Queue接口

Collection接口

List接口

  • ArrayList：动态数组
  • LinkedList：双向链表
  • Vector：开销大，线程安全的动态数组

Set接口

  • HashSet：基于哈希表，无序，不允许重复
  • LinkedHashSet：基于链表和哈希表，维持插入顺序，不允许重复
  • TreeSet：基于红黑树，有序不重复

Queue接口

  • PriorityQueue：优先队列
  • LinkedList：当队列使用，先进先出

Map接口

  • HashMap：键值对无序，不允许键重复插入
  • LinkedHashMap：基于链表和哈希表，不允许键值重复
  • TreeMap：基于红黑树，键值对有序
  • Hashtable：线程安全哈希表，不允许键或值为null
  • ConcurrentHashMap：线程安全，适合高并发环境，不允许键或值为null

什么是Java中的双亲委派模型

核心流程

双亲委派模型指的是对某一个类类加载的时候，会先委派给其父加载器去加载，如果发现父加载器无法加载，再由当前类加载器加载(这个委派的过程是逐层向上的)

分类

JDK9前，Java主要类加载器可以分为3类

1）启动类加载器

2）扩展类加载器

3）应用程序加载器

加载器的查找流程一般是从应用程序加载器出发，先委派给扩展类加载器，最终到启动类加载器

双亲委派机制的好处

可以拿java.lang.Object这个Java中最大的父类来讲，加载它最终是委派给启动类加载器去加载/lib/rt.jar内部的Object到JVM

如果别人伪造了一个java.lang.Object，如果根据双亲委派机制，最终只会加载我们rt.jar中的内容

所以说好处主要包含

1）可以避免重复加载类，Object类只能由根加载器加载，防止加载恶意代码

2）确保一个类在JVM中只能加载一次

有没有违反双亲委派的例子？

答案是JDBC

JDBC接口由类库定义，但是实现是各大数据库厂商提供的jar包，启动类加载器是找不到这个类的，需要使用应用程序加载器，违反自下而上

Java的类加载过程是什么？

类加载的本质

本质是为了将类加载到JVM中，先将其转化为二进制流后加载到内存中，经过解析转化为可以使用的class类

加载流程主要分为

1）加载：通过类加载器将类文件加载到内存中，生成class对象

2）连接：又可以分为三个字阶段验证、准备、解析，校验class文件是否合法、为静态变量划分内存空间

3）初始化：整合静态变量和静态代码块初始化操作执行

JDK8有哪些新特性

新增特性

  • Lambda表达式

  • Stream流式接口

  • Optional类

  • 并发类：CompletableFuture、StampedLock

    拓展：HashMap、ConcurrentHashMap，可以和面试官拉扯

元空间相比于永久代

元空间很灵活、扩展性强，它可以调整固定大小限制，数据也可以存储本地内存，不受Java堆限制，满足各种应用程序需求

JDK动态代理和CGLIB代理区别？

JDK动态代理基于接口，代理类需要定义接口

CGLIB基于ASM字节码生成工具，通过继承，生成目标类的字类来实现代理类

观察上述代码，我们就发现静态代理是在编译的时候就进行了代理操作，而动态代理是在运行的时候进行的

什么是Java中的SPI？

这里可以拿API和SPI进行对比

SPI

接口和调用方已经写好，开发者可以扩展

API

接口和实现的方法已经写好，别人可以直接拿来用里面方法实现方法

数组和链表在Java中的区别

存储结构

数组属于连续的内存块，每一块大小固定，想要改变数组大小需要重新分配内存

链表基于节点，可以动态插入和删除，内存不连续但可以动态扩展

访问速度

数组通过索引直接访问，时间复杂度为O(1)

链表由于内存中不连续，访问时间为O(n)

操作

数组插入和删除时间复杂度为O(n)

链表可以在O(1)时间复杂度内插入数据

应用场景

数组针对于大小固定、快速访问，实现缓存等数据结构

链表针对于操作，例如栈、队列等

什么是Java的CAS操作？

CAS(Compare And Swap)原子操作，用来比较内存中某一个值是否和预期值相同

原理

  • Compare：检查内存中某个值是否和预期值相同
  • Swap：如果相同，更新内存中的值
  • 失败：不相等，说明别的线程修改了该值，CAS操作失败，重试直到成功

举例说明

假如我们有一个累加的需求，如果这个值等于1，那么就转化为2，如果这个值等于2，那么我们转化为3

在多线程的条件下，两个线程同时比较值是否等于1，发现都满足条件，最终结果都是2，但是我们想要的是3

这种情况可以加锁，但是浪费资源

我们可以在硬件层面封装比较和交换成一个指令，保证原子性，这个指令就是CAS

CAS需要三个变量：老的值，变量内存地址，新的值

指令通过变量地址拿到值，进行比较，如果相等替换否则不换

优缺点

优点

  • 无锁并发
  • 原子性

缺点

  • ABA问题，A变成B后又变回A
  • 自旋开销
  • 单变量限制

ABA问题解决方案

ABA指的是变量从A变成B又变成A，CAS检测不到这种变化

常见解决方法：引入版本号和时间戳，更新变量的同时更新版本号，检测变量变化

Java中AtomicStampedReference内部Pair封装了引用和版本号，同时使用volatile修饰保证可见

CAS内部添加了版本号对比

Java还提供了AtomicMarkableReference，与上面的Java类类似，内部新增布尔类型的mark变量，用来表示数据是否修改过

CAS在Java中的实现

CAS是由Unsafe类提供的，使用native修饰，通过JNI调用底层硬件实现的

并发包中Atomic类封装了CAS操作，保证线程的原子操作
